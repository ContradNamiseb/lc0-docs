85// Interface for search stopper.
86// Note that:
87// 1. Stoppers are shared between all search threads, so if stopper has mutable
88// varibles, it has to think about concurrency (mutex/atomics)
89// (maybe in future it will be changed).
90// 2. IterationStats and StoppersHints are per search thread, so access to
91// them is fine without synchronization.
92// 3. OnSearchDone is guaranteed to be called once (i.e. from only one thread).
93class SearchStopper {
94 public:
95 virtual ~SearchStopper() = default;
96 // Question to a stopper whether search should stop.
97 // Search statistics is sent via IterationStats, the stopper can optionally
98 // send hints to the search through StoppersHints.
99 virtual bool ShouldStop(const IterationStats&, StoppersHints*) = 0;
100 // Is called when search is done.
101 virtual void OnSearchDone(const IterationStats&) {}
102};
